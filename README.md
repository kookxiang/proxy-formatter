# Proxy Provider 使用说明

本项目是一个代理配置处理工具，支持多种操作，可对代理配置进行灵活的修改和处理。

## 使用方法

1. **创建配置文件**
   创建一个文本文件，每行定义一个操作及其参数。例如：
   ```text
   fetch https://example.com/proxies.yaml
   include /US/
   replace oldName newName
   ```

2. **启动服务**
   运行程序时，指定配置文件所在的目录：
   ```bash
   ./proxy-provider -dir /path/to/config
   ```

3. **访问 HTTP 服务**
   程序会启动一个 HTTP 服务，默认监听 `8080` 端口。通过访问对应的文件路径，程序会按照配置文件中的操作逐一执行。例如：
   ```
   http://localhost:8080/config.txt
   ```

4. **日志与缓存**
   - 配置文件目录通过 `-dir` 参数指定。
   - 缓存目录通过 `-cache-dir` 参数指定，默认为系统临时目录下的 `cache` 文件夹。

---

## 支持的操作

### 1. `emoji`
- **描述**: 移除代理名称中的 emoji 标志。
- **参数**: 无。

---

### 2. `fetch`
- **描述**: 从指定的 URL 获取代理数据并缓存响应。
- **参数**:
  - `<url>`: 要获取数据的 URL。

---

### 3. `include` / `exclude`
- **描述**: 根据模式过滤代理。
- **参数**:
  - `<include|exclude>`: 指定是包含还是排除匹配模式的代理。
  - `<pattern>`: 匹配的模式（如果用 `/` 包裹，则支持正则表达式）。

---

### 4. `freeze`
- **描述**: 将所有代理标记为冻结状态。
- **参数**: 无。

---

### 5. `header`
- **描述**: 修改请求头。
- **参数**:
  - `<key>`: 请求头的键。
  - `[value]`: 请求头的值（可选；如果省略，则删除该请求头）。

---

### 6. `prefix` / `suffix`
- **描述**: 为代理名称添加或移除前缀或后缀。
- **参数**:
  - `<content>`: 要添加或移除的前缀或后缀内容。

---

### 7. `proxy`
- **描述**: 设置代理地址。
- **参数**:
  - `[address]`: 代理地址（可选；如果省略，则清除代理）。

---

### 8. `replace`
- **描述**: 替换代理名称中的字符串。
- **参数**:
  - `<from>`: 要替换的字符串。
  - `<to>`: 替换后的字符串。

---

### 9. `resolve`
- **描述**: 为代理服务器解析 DNS。
- **参数**:
  - `[dns]`: 使用的 DNS 服务器（可选；默认为 `119.29.29.29`）。
